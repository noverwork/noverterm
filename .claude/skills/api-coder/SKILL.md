# API Coder - Rust Backend to TypeScript Contract Guide

API contract rules for the Rust backend and Svelte frontend in this monorepo.

- **Project** = local rule for this repo
- **Tooling** = `ts-rs` behavior or command requirement
- **Common** = broadly accepted API contract practice

## Core Rule

Rust backend API request/response types that cross the HTTP boundary must live in `packages/shared`, not `packages/backend`.

This keeps TypeScript generation independent from backend implementation dependencies such as Diesel/Postgres `libpq`.

## Quick Reference

### File Locations

| Artifact | Location | Rule |
| --- | --- | --- |
| Rust API contract types | `packages/shared/src/lib.rs` | Shared crate owns request/response DTOs |
| Axum handlers | `packages/backend/src/**/routes.rs` | Import DTOs from `shared` |
| Generated TS bindings | `packages/desktop/ui/src/bindings/*.ts` | Generated by `ts-rs`, do not edit |
| Tauri command bindings | `packages/desktop/ui/src/bindings.ts` | Generated by `tauri-specta`, separate from API types |

### Naming

| Thing | Convention | Example | Source |
| --- | --- | --- | --- |
| Rust struct/enum | `PascalCase` | `HostGroupWriteRequest` | Rust convention |
| TypeScript type name | `PascalCase` | `HostGroupWriteRequest` | Matches Rust type |
| Generated TS filename | `kebab-case` | `host-group-write-request.ts` | Project rule |
| Rust fields / JSON keys | `snake_case` unless serde overrides | `refresh_token` | Current API convention |

## Required Pattern

### Define API DTOs in `shared`

```rust
use serde::{Deserialize, Serialize};
use ts_rs::TS;

#[derive(Debug, Clone, Deserialize, TS)]
#[ts(export, export_to = "host-group-write-request.ts")]
pub struct HostGroupWriteRequest {
    pub name: String,
}

#[derive(Debug, Clone, Serialize, Deserialize, TS)]
#[ts(export, export_to = "auth-response.ts")]
pub struct AuthResponse {
    pub access_token: String,
    pub refresh_token: String,
    pub access_token_expires_at: chrono::DateTime<chrono::Utc>,
    pub email: String,
}
```

### Import DTOs in backend routes/services

```rust
use shared::{AuthResponse, LoginRequest};

async fn login(
    axum::extract::State(state): axum::extract::State<AppState>,
    axum::Json(request): axum::Json<LoginRequest>,
) -> Result<axum::Json<AuthResponse>, (axum::http::StatusCode, String)> {
    state.auth_service.login(request).await.map(axum::Json)
}
```

### Generate TypeScript bindings

```bash
TS_RS_EXPORT_DIR=packages/desktop/ui/src/bindings cargo test -p shared
```

Expected output shape:

```typescript
// host-group-write-request.ts
export type HostGroupWriteRequest = { name: string };
```

## Dependencies

`packages/shared/Cargo.toml` should own `ts-rs` and any type feature support needed by exported DTOs.

```toml
ts-rs = { version = "10", features = ["serde-compat", "chrono-impl", "uuid-impl"] }
chrono = { version = "0.4", features = ["serde"] }
```

Only keep features that are needed by exported shared types.

## Forbidden Patterns

```rust
// ❌ Do not define HTTP API DTOs only inside backend route modules.
#[derive(Debug, Deserialize, ts_rs::TS)]
#[ts(export, export_to = "host-write-request.ts")]
struct HostWriteRequest {
    name: String,
}

// ❌ Do not rely on `cargo test -p backend` to generate API type bindings.
// It can fail before export when backend links Diesel/Postgres/libpq.
```

## Key Rules

1. **[Project]** Put HTTP API DTOs in `packages/shared` when frontend needs the type.
2. **[Project]** Keep generated TypeScript filenames in kebab-case with explicit `export_to`.
3. **[Project]** Keep TypeScript type names matching Rust type names in PascalCase.
4. **[Project]** Do not edit generated `packages/desktop/ui/src/bindings/*.ts` files manually.
5. **[Tooling]** Use both `export` and `export_to`; `export_to` alone does not create export tests.
6. **[Tooling]** Generate API bindings with `cargo test -p shared`, not `cargo test -p backend`.
7. **[Common]** Request DTOs usually derive `Deserialize`; response DTOs derive `Serialize`; shared or test-used DTOs may derive both.
8. **[Project]** Backend route-local internal-only types may remain in `backend`, but any frontend-facing API shape must move to `shared`.

## Workflow

1. Identify the request/response shape used by an Axum handler.
2. If frontend consumes it, define or move the type into `packages/shared/src/lib.rs`.
3. Add `#[derive(TS)]` and `#[ts(export, export_to = "kebab-case-name.ts")]`.
4. Update backend route/service imports to use `shared::{...}`.
5. Remove duplicate local DTO definitions from backend modules.
6. Run:

```bash
cargo check -p shared -p backend
TS_RS_EXPORT_DIR=packages/desktop/ui/src/bindings cargo test -p shared
```

7. Verify generated filenames are kebab-case and imports inside generated files point to kebab-case paths.

## Troubleshooting

### Type file did not generate

Check that the Rust type uses both attributes:

```rust
#[derive(TS)]
#[ts(export, export_to = "my-type.ts")]
```

`#[ts(export_to = "my-type.ts")]` without `export` changes the output path but does not generate the automatic export test.

### Backend export fails with `ld: library 'pq' not found`

Do not generate API contracts from `backend`. Move frontend-facing DTOs to `shared` and run `cargo test -p shared` instead.

### TypeScript type name differs from filename

That is expected. The filename is kebab-case, but the TypeScript type name remains PascalCase to match the Rust struct or enum.
