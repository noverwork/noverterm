# Tauri Frontend Integration Reference

## Project Setup

This project uses **SvelteKit** as the frontend, located at `packages/desktop/ui/`.

### Configuration

**`tauri.conf.json` build section:**
```json
{
  "build": {
    "beforeDevCommand": {
      "cwd": "ui",
      "script": "npm run dev"
    },
    "devUrl": "http://localhost:1420",
    "beforeBuildCommand": {
      "cwd": "ui",
      "script": "npm run build"
    },
    "frontendDist": "ui/dist"
  }
}
```

- **[Project]** Frontend lives in `ui/` subdirectory, not at repo root
- **[Project]** `devUrl` port is `1420` — vite config must match
- **[Project]** `frontendDist` points to `ui/dist` (Vite's default output)

### SvelteKit Adapter

```javascript
// svelte.config.js
import adapter from '@sveltejs/adapter-static';
import { vitePreprocess } from '@sveltejs/vite-plugin-svelte';

export default {
  preprocess: vitePreprocess(),
  kit: {
    adapter: adapter({
      fallback: 'index.html'
    })
  }
};
```

- **[Official]** MUST use `@sveltejs/adapter-static` for Tauri — not the default node adapter
- **[Official]** `fallback: 'index.html'` enables SPA mode (client-side routing)

### Vite Configuration

```javascript
// vite.config.js
import { defineConfig } from 'vite';

export default defineConfig({
  clearScreen: false,
  server: {
    port: 1420,
    strictPort: true,
    watch: {
      ignored: ['**/src-tauri/**'],
    },
  },
});
```

- **[Official]** `strictPort: true` — fail if port is unavailable (Tauri needs a fixed port)
- **[Official]** Ignore `src-tauri` from vite's file watcher

## Invoking Commands

### Generated Bindings (Preferred)

This project uses `tauri-specta` to generate TypeScript bindings:

```typescript
// src/bindings.ts — auto-generated by export_types binary
import { invoke as TAURI_INVOKE } from "@tauri-apps/api/core";

export type Result<T, E> =
  | { status: "ok"; data: T }
  | { status: "error"; error: E };

// Commands are typed functions wrapping invoke()
```

- **[Project]** NEVER manually type `invoke()` calls — use generated bindings
- **[Project]** Bindings are regenerated by running the `export-types` binary
- **[Project]** Bindings path: `packages/ui/src/bindings.ts`

### Manual Invoke (When Needed)

```typescript
import { invoke } from '@tauri-apps/api/core';

// Simple command
const greeting = await invoke('greet', { name: 'World' });

// Async command with error handling
try {
  const result = await invoke('bootstrap_restore');
  console.log(result);
} catch (error) {
  console.error('Command failed:', error);
}
```

- **[Official]** Command name matches Rust function name exactly (snake_case)
- **[Official]** Arguments are passed as an object with keys matching Rust parameter names
- **[Official]** `invoke()` returns a Promise

## Events

```typescript
import { listen, emit } from '@tauri-apps/api/event';

// Listen
const unlisten = await listen('event-name', (event) => {
  console.log(event.payload);
});

// Emit
await emit('event-name', { key: 'value' });

// Cleanup
unlisten();
```

## Environment Variables

```javascript
// vite.config.js
envPrefix: ['VITE_', 'TAURI_ENV_*'],
```

- **[Official]** Only `VITE_*` and `TAURI_ENV_*` prefixed variables are exposed to the frontend
- **[Official]** `TAURI_ENV_DEBUG`, `TAURI_ENV_PLATFORM` are set by Tauri during build
